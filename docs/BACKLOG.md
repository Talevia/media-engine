- **debt-cache-invalidate-asset-graph-hash** — `src/api/render.cpp:46-47` documents this: `me_cache_invalidate_asset` was originally written when DiskCache keys had shape `<asset_hash>:<src_t>` so prefix-matching the asset hash dropped derived frames; the move to graph-content-hash keys (`g:<hash>`) made that prefix match a no-op. Confirmed empirically by `examples/12_cache_invalidate`: after `invalidate_asset`, `entry_count` (AssetHashCache) drops as expected but DiskCache entries derived from the same asset still serve the next render — `miss_count` does not climb. Hosts that need real eviction call `me_cache_clear`. **方向：** maintain a side index `asset_hash → set<graph_hash>` populated on every DiskCache put and consulted by invalidate_asset (option (b) per the original bullet — O(1) per-asset eviction with a write-amplification step). Defer implementation until M11+ workloads supply DiskCache size measurements. Milestone §M6-debt (cross)，Rubric §5.4。
- **inverse-tonemap-hable-impl** — Cycle 24 landed `Linear` in `src/compose/inverse_tonemap_kernel.cpp:apply_inverse_tonemap_inplace`; `Hable` algo still returns ME_E_UNSUPPORTED at line 70 (gated by an explicit early-return). Real Hable inverse needs linear-light float buffers: the curve `f(x) = ((x*(A*x+C*B)+D*E)/(x*(A*x+B)+D*F))-E/F` is invertible only on the [0, white_scale] domain, and byte-domain inverse rounding loses enough precision that the round-trip fails VISION §3.1 byte-identity. **方向：** when a 16-bit (or RgbaF32) working buffer arrives in M11+, replace the Hable UNSUPPORTED branch with the proper linear-light inverse: read SDR byte → linearize via inverse-sRGB-EOTF → numerical inverse of Hable curve via Newton–Raphson on the [0, white_scale] domain → encode to PQ / HLG / linear-light HDR buffer. Until then leave UNSUPPORTED as the deterministic answer for Hable. Milestone §M11-followup，Rubric §5.2。
- **face-sticker-impl** — Cycle 30 landed `EffectKind::FaceSticker` + `FaceStickerEffectParams` + JSON loader dispatch + STUB-marked `apply_face_sticker_inplace` kernel that returns ME_E_UNSUPPORTED for any input (`src/compose/face_sticker_kernel.cpp:38`). M11 exit criterion 7+8 (docs/MILESTONES.md:140-145) requires the actual landmark-driven sticker overlay to ship. **方向：** the impl needs three landed prerequisites first: (1) inference runtime that consumes the Landmark asset (downstream M11 cycles, blocked on cycle 29's model-fetcher being wired to a CoreML / ONNX adapter); (2) sticker-decoder kernel for PNG / WebP via libavformat probe (mirrors `me_thumbnail_png` shape but on a pre-existing image asset); (3) compose-graph stage that overlays the sticker onto the RGBA8 frame at the resolved landmark anchor with affine scale + offset (overlaps with `RenderAffineBlit` but per-frame landmark-time-coupled). When all three land, replace the UNSUPPORTED return with: read landmark[frame_t] → compute anchor + bbox → load sticker via decoder → affine-blit at anchor. Milestone §M11，Rubric §5.1。
- **face-mosaic-impl** — Cycle 34 landed `EffectKind::FaceMosaic` + `FaceMosaicEffectParams { landmark_asset_id, block_size_px, kind: Pixelate/Blur }` + JSON loader dispatch + STUB-marked `apply_face_mosaic_inplace` kernel that returns ME_E_UNSUPPORTED for any input (`src/compose/face_mosaic_kernel.cpp:30`). The actual mosaic / blur math is deterministic byte-arithmetic; the block is the upstream wiring (same prerequisites as `face-sticker-impl`: inference runtime + landmark stream resolution + per-frame bbox computation). **方向：** when the M11 inference runtime cycles land + the landmark stream is resolvable per-frame, replace the UNSUPPORTED return with: read landmark[frame_t] → compute bbox → for Pixelate: clamp bbox to image, walk in `block_size_px` tiles, write per-channel mean × block; for Blur: same walk but with a box-filter pass at radius `block_size_px / 2`. Both are CPU-deterministic — no parallelism, no SIMD, no rand. Milestone §M11，Rubric §5.1。
- **body-alpha-key-impl** — Cycle 35 landed `EffectKind::BodyAlphaKey` + `BodyAlphaKeyEffectParams { mask_asset_id, feather_radius_px, invert }` + JSON loader dispatch + STUB-marked `apply_body_alpha_key_inplace` kernel that returns ME_E_UNSUPPORTED (`src/compose/body_alpha_key_kernel.cpp:28`). The actual mask-key math is deterministic byte-arithmetic; the block is upstream wiring (same prerequisites as `face-sticker-impl` / `face-mosaic-impl`: inference runtime + per-frame mask-asset resolution). **方向：** when M11 inference runtime cycles land + the Mask asset is resolvable per-frame as an alpha buffer, replace the UNSUPPORTED return with: read mask[frame_t] → optionally invert (`mask = 255 - mask`) → optionally box-blur the alpha edge by `feather_radius_px` → write `(mask × input.alpha) / 255` into output.alpha. CPU-deterministic. Milestone §M11，Rubric §5.1。
