## P0（必做，阻塞当前 milestone）

- **inference-runtime-factory-impl** — `me::inference::Runtime` exists at `src/inference/runtime.hpp:91` as the abstract base for `CoreMLRuntime` / `OnnxRuntime`; `me::inference::load_model_blob` (`src/inference/model_loader.hpp:84`) loads + validates model bytes. `grep -rn 'make_runtime\|RuntimeFactory' src/ tests/` returns empty — no factory glues the two together. Without a factory, production stages have no canonical way to go from `(model_id, model_version, quantization)` to a callable `Runtime*` that's been license-checked + content-hash-validated. M11 §138 production-call-site half is blocked on this. **方向：** add `me::inference::make_runtime_for_model(me_engine*, model_id, model_version, quantization, std::unique_ptr<Runtime>* out, std::string* err)` in `src/inference/runtime_factory.{hpp,cpp}`. Calls `load_model_blob` (which consults the engine's `loaded_models` cache + license whitelist + content_hash). Picks runtime impl by platform availability (CoreML on macOS when ME_HAS_COREML, ONNX otherwise when ME_HAS_ONNX_RUNTIME). Caches the constructed Runtime on the engine indexed by the same identity tuple so repeated calls return the same instance. Unit test mocks both the loader fetch + a synthetic Runtime to verify factory behavior. Milestone §M11，Rubric §5.1。
- **landmark-resolver-runtime-mode-impl** — `src/compose/landmark_resolver.{hpp,cpp}` ships a file-mode resolver only (`resolve_landmark_bboxes_from_file`); `grep -rn 'run_cached\|Runtime::run' src/compose` returns empty. M11 §137 ("推理 asset 走 §3.3 contentHash 缓存") and §138 ("engine 校验 content_hash + license 白名单") both require a production caller of `Runtime::run` via `run_cached`. **方向：** add `resolve_landmark_bboxes_runtime(me_engine*, asset_uri_or_model_ref, frame_t, &bboxes, &err)` in landmark_resolver.cpp. Calls `make_runtime_for_model` (P0 above) → `me::inference::run_cached` with the input frame as the inference input → parses the runtime's output tensor into the bbox span. Wire face_sticker_stage / face_mosaic_stage to choose runtime-vs-file based on whether the asset's URI has `model:` prefix vs file path. End-to-end test exercises one frame through BlazeFace via run_cached, asserts the cache hits on second call. Milestone §M11，Rubric §5.1。

## P1（强烈建议，跨 milestone debt）

- **talevia-jvm-wrapper** — M7 exit criterion "在 talevia 内建 platform-impls/video-media-engine-jvm". Cross-repo task — needs talevia-side work; in this repo `bindings/jni/` is ready (cycle 28, commit 89b7275). **方向：** copy JNI wrapper to talevia tree + build.gradle.kts + replace shell-out FFmpeg in passthrough tests. This repo cycle can only prepare the JNI wrapper; integration lives in talevia. Milestone §M7，Rubric §5.5。
- **inverse-tonemap-hable-impl** — Cycle 24 landed `Linear` in `src/compose/inverse_tonemap_kernel.cpp:apply_inverse_tonemap_inplace`; `Hable` algo still returns ME_E_UNSUPPORTED at line 70 (gated by an explicit early-return). Real Hable inverse needs linear-light float buffers: the curve `f(x) = ((x*(A*x+C*B)+D*E)/(x*(A*x+B)+D*F))-E/F` is invertible only on the [0, white_scale] domain, and byte-domain inverse rounding loses enough precision that the round-trip fails VISION §3.1 byte-identity. **方向：** when a 16-bit (or RgbaF32) working buffer arrives in M11+, replace the Hable UNSUPPORTED branch with the proper linear-light inverse: read SDR byte → linearize via inverse-sRGB-EOTF → numerical inverse of Hable curve via Newton–Raphson on the [0, white_scale] domain → encode to PQ / HLG / linear-light HDR buffer. Until then leave UNSUPPORTED as the deterministic answer for Hable. Milestone §M11-followup，Rubric §5.2。
- **mask-resolver-runtime-mode-impl** — Sibling of landmark-resolver-runtime-mode-impl (P0). `src/compose/mask_resolver.{hpp,cpp}` ships only `resolve_mask_alpha_from_file` (file-mode JSON+base64 sidecar). `grep -rn 'run_cached\|Runtime::run' src/compose/mask_resolver.cpp` returns empty. body_alpha_key_stage's runtime mode is the second M11 §137/§138 production call site (after landmark resolver). **方向：** add `resolve_mask_alpha_runtime(me_engine*, asset_or_model_ref, frame_t, &mask_w, &mask_h, &alpha, &err)`. Calls `make_runtime_for_model` for SelfieSegmentation → `run_cached` with the input frame → parses the alpha mask tensor into `out_alpha`. Wire body_alpha_key_stage to dispatch file-vs-runtime based on the asset's URI shape. End-to-end test exercises SelfieSegmentation via run_cached on a fixture frame. Milestone §M11，Rubric §5.1。
- **selfie-segmentation-ship-path-test** — M11 §139 exit criterion at `docs/MILESTONES.md:139` requires "至少 2 个 ship-path model 跑通". `tests/test_inference_blazeface.cpp` exists (env-gated). `grep -rn 'SelfieSegmentation' tests/ src/` returns empty — the second model for §139 has no test. **方向：** new `tests/test_inference_selfie_segmentation.cpp` mirroring the BlazeFace pattern: env-var-gated host-supplied model bytes (`ME_TEST_SELFIE_SEG_MODEL` env var pointing at a .onnx / .mlmodel file), feed a known fixture image through `me::inference::Runtime::run`, assert the output mask's pixel-domain mean alpha is within a small ε of an offline-computed reference. Pair with `examples/14_ml_face_landmark`-shape demo if needed for ship-path documentation. Milestone §M11，Rubric §5.1。
- **inference-runtime-factory-test** — Pairs with `inference-runtime-factory-impl` (P0). Once the factory lands, it needs unit coverage independent of any specific model. **方向：** new `tests/test_inference_runtime_factory.cpp`. Milestone §M11，Rubric §5.1。

## P2（未来，当前 milestone 不挤占）

- **encode-hevc-main10-kvazaar-source-build** — `src/io/kvazaar_hevc_encoder.cpp:88` calls `kvz_api_get(8)` exclusively; the homebrew Kvazaar bottle's `kvazaar.h` defaults to `KVZ_BIT_DEPTH 8` and the prebuilt library doesn't expose a 10-bit `kvz_pixel` ABI. `cfg->input_bitdepth = 10` is accepted by `encoder_open` but the `picture_alloc` buffer geometry stays 8-bit. M10 exit criterion 3 (docs/MILESTONES.md:120) was satisfied by the M10 SW HEVC fallback shipping Main 8-bit only; HDR Main 10 stays on the VideoToolbox HW path. This bullet is the future-improvement to bring SW HEVC up to Main 10 for cross-host HDR. **方向：** replace `pkg_check_modules(kvazaar)` with `FetchContent_Declare(kvazaar URL ...)` + `cmake -DKVZ_BIT_DEPTH=10` so the engine links its own 10-bit-built libkvazaar.a; rewrite the encoder TU's plane-copy loops to pass uint16_t buffers. Add a fixture YUV420P 10-bit gray frame and a round-trip test that decodes the produced HEVC Main 10 and verifies pixel domain matches within ε. Note: Kvazaar's build system is autotools (no native CMake); FetchContent will need ExternalProject_Add wrapping. Milestone §M10-followup，Rubric §5.2。
- **debt-codec-dispatch-table-unification** — Cycle 48 landed `me::orchestrator::resolve_codec_selection` mapping codec strings → `me_video_codec_t` enum (`src/orchestrator/codec_resolver.cpp:18-32`).
