# effect shaders — source + compiled bytecode

bgfx effects (color-correct / future blur / future LUT) load their shaders from
precompiled bytecode headers generated by `shaderc`. Shader source lives next to
the `.sc` files in this directory; compiled output lives in `generated/`.

## Regenerating

`shaderc` is the bgfx shader compiler — built from the bgfx.cmake fetchcontent'd
source when `-DBGFX_BUILD_TOOLS=ON` (off by default in our build). Regenerate
the headers on a dev machine after changing a `.sc` file:

```sh
cmake -B build-gpu -DME_WITH_GPU=ON -DBGFX_BUILD_TOOLS=ON \
    -DBGFX_BUILD_TOOLS_GEOMETRY=OFF -DBGFX_BUILD_TOOLS_TEXTURE=OFF \
    -DBGFX_BUILD_EXAMPLE_COMMON=OFF
cmake --build build-gpu --target shaderc

SHADERC=build-gpu/_deps/bgfx_cmake-build/cmake/bgfx/shaderc
SHADER_INC=build-gpu/_deps/bgfx_cmake-src/bgfx/src
SHDIR=src/effect/shaders
OUT=$SHDIR/generated

$SHADERC -f $SHDIR/vs_fullscreen.sc     -o $OUT/vs_fullscreen_mtl.bin.h \
    --type v --platform osx -p metal --bin2c vs_fullscreen_mtl \
    -i $SHADER_INC --varyingdef $SHDIR/varying.def.sc

$SHADERC -f $SHDIR/fs_color_correct.sc  -o $OUT/fs_color_correct_mtl.bin.h \
    --type f --platform osx -p metal --bin2c fs_color_correct_mtl \
    -i $SHADER_INC --varyingdef $SHDIR/varying.def.sc
```

Commit the regenerated `.bin.h` alongside the `.sc` change.

## Per-platform coverage

Today only Metal (`-p metal --platform osx`) is generated — it's the only
renderer our dev / CI environment exercises. Cross-platform bringup (Vulkan,
D3D12, OpenGL) extends this step with additional `--profile spirv` /
`--profile s_6_0` / `--profile 430` invocations, each producing a new
`<shader>_<backend>.bin.h`; `color_correct_effect.cpp` picks the right one
based on `bgfx::getRendererType()`.
