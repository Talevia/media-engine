# media-engine static library size budget — VISION §5.7-5 guard.
#
# tools/check_lib_size.sh reads the size of
# `$<TARGET_FILE:media_engine>` and asserts ≤ the per-config ceiling
# below. ctest "lib_size_budget" runs the script.
#
# Why per-config: Debug carries DWARF + assertions; Release strips
# both. A single number would either be permissive enough that
# release inflation hides, or tight enough that Debug always fails.
# Per-config keeps both regimes honest.
#
# Bumping these REQUIRES a commit body explaining the source of
# growth (new dependency, new GPU path linked unconditionally,
# generated tables). Silent inflation = link-graph debt that we
# don't see until the Android APK or iOS bundle bloats.
#
# Baselines as of M9 inception (cycle establishing this guard).
# This repo's top-level CMakeLists.txt defaults CMAKE_BUILD_TYPE to
# Release when unset, so `cmake -B build -S .` hits the Release row;
# Debug only fires on explicit `-DCMAKE_BUILD_TYPE=Debug`.
#
# Measured on dev box (cycle establishing this guard):
#   Release build/src/libmedia_engine.a ≈ 2.3 MB
# Debug not yet measured — ceiling left at the same 4 MB pending an
# explicit Debug build.
#
# Format: <CONFIG>=<bytes>
# Lines starting with # are comments. Unknown configs are skipped by
# check_lib_size.sh (LENIENT — see comment there). Tighten as CI
# stabilises and the per-config baseline variance is known.

Debug=4194304
Release=4194304
RelWithDebInfo=4194304
MinSizeRel=2097152

# M11 ML inference runtime budget rows — sentinel scaffold from
# cycle 27 (ml-inference-cmake-option). Active only when CMake is
# configured with -DME_WITH_INFERENCE=ON; tests/CMakeLists.txt
# appends the `_with_inference` suffix to $<CONFIG> in that case.
# Sentinel ceilings (16 MB Release / 24 MB Debug) generous enough
# for the planned CoreML adapter + ONNX-runtime CPU FP32 binding +
# the engine's own bytes. Will tighten once the M11 runtime cycles
# land and an actual measurement is recorded. Per the bumping
# rule above, any ceiling adjustment requires a commit body
# explaining the source of growth.
Debug_with_inference=25165824
Release_with_inference=16777216
RelWithDebInfo_with_inference=16777216
MinSizeRel_with_inference=8388608

# M10 SW HEVC encoder budget rows — `ME_WITH_KVAZAAR=ON` adds the
# `_with_kvazaar` suffix; combined with ME_WITH_INFERENCE the
# suffix becomes `_with_inference_with_kvazaar`.
#
# Measurements on dev box (cycle establishing the rows):
#   Release base                          : 2,824,720 B
#   Release with kvazaar                  : 2,842,240 B  (+17 KB; kvazaar dyn-linked)
#   Release with inference + kvazaar      : 2,961,872 B  (+137 KB over base)
# Static archive growth is small because libkvazaar.dylib is
# dynamic; the new bytes are the kvazaar_hevc_encoder.o object
# file plus its small RAII state. The ceiling is set to match the
# parallel non-kvazaar row so future inflation (a bigger encoder
# adapter, a new fallback path, generated tables) trips the guard.
